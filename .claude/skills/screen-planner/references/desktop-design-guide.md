# 데스크탑 화면 설계 가이드

Screen Planner가 데스크탑(웹) 플랫폼 화면을 설계할 때 참조하는 원칙과 패턴.

---

## 1. 핵심 원칙: 정보구조 ≠ 화면구성

데스크탑에서는 정보구조의 여러 단위를 하나의 화면 안에 병렬 배치할 수 있다. 모바일에서 5화면으로 풀어야 할 정보를, 사이드바 + 메인 영역 + 패널 조합으로 1-2화면에 담을 수 있다.

**판단 기준:**
- 사용자가 정보를 교차 참조해야 하면 → 같은 화면, 다른 영역에 배치
- 작업 맥락이 완전히 바뀌면 → 별도 페이지
- 화면 수가 적어지는 대신 화면 내 복잡도가 올라감 — 영역 간 시각적 위계가 핵심
- "한 화면에 다 넣을 수 있다"와 "넣어야 한다"는 다름. 밀도가 높을수록 시각적 구조가 명확해야 함

---

## 2. 페이지 레벨 레이아웃 패턴

개별 컴포넌트(모달, 탭 등) 이전에, **페이지 전체를 어떤 구조로 잡을 것인가**를 결정하는 매크로 레벨 패턴. 화면 설계 시 가장 먼저 선택해야 할 뼈대.

### 스크린 패턴 (화면의 구조적 뼈대)

| 패턴 | 구조 | 적합한 경우 |
|------|------|------------|
| **마스터-디테일** | 좌측 목록 + 우측 상세. 목록에서 선택하면 상세가 바뀜 | 이메일, 이슈 트래커, 파일 탐색 — 여러 항목을 빠르게 훑으며 비교 |
| **대시보드** | 위젯/카드들의 그리드 배치. 각 위젯이 독립적 데이터를 표시 | KPI 모니터링, 분석 도구 — 한눈에 전체 상황 파악 |
| **캔버스/팔레트** | 좌측이나 상단에 도구 모음 + 넓은 작업 영역 | 디자인 도구, 다이어그램 편집기, 코드 에디터 — 자유로운 조작이 핵심 |
| **컬럼 브라우즈** | 여러 컬럼을 가로로 나열하며 계층을 탐색 (Finder 스타일) | 깊은 트리 구조 탐색 — 카테고리 > 하위카테고리 > 항목 |
| **스프레드시트** | 행과 열의 테이블. 셀 단위 편집 가능 | 대량 데이터 입력/편집, 관리자 도구 — 밀도 높은 데이터 작업 |
| **위자드 (스텝)** | 순차적 단계를 거치는 선형 흐름. 프로그레스 표시 | 복잡한 설정, 온보딩, 체크아웃 — 한 번에 하기 어려운 작업을 분해 |
| **검색/결과** | 검색 입력 + 필터 + 결과 목록 | 탐색 중심 서비스 — 사용자가 원하는 것을 찾아가는 흐름 |
| **필터 데이터셋** | 좌측/상단 필터 + 결과 영역. 필터 변경 시 즉시 반영 | 이커머스 상품 목록, 데이터 탐색 — 조건을 좁혀가며 찾기 |
| **인터랙티브 모델** | 핵심 객체(지도, 캘린더, 차트)가 화면 중심. 주변에 조작 도구 | 지도 기반 서비스, 캘린더, 데이터 시각화 — 멘탈모델과 직결되는 객체 |
| **패러렐 패널** | 여러 정보 블록을 병렬로 배치 (동시 또는 하나씩 전환) | 비교 뷰, 상호의존적 정보 블록 — 교차 참조가 필요한 경우 |
| **Q&A / 가이드** | 질문에 답하면 다음 질문으로. 최종적으로 추천/결과 도출 | 추천 엔진, 진단 도구, 맞춤형 제안 — 사용자 입력 기반 결과 |

### 페이지 레이아웃 패턴 (콘텐츠 배치의 시각적 구조)

| 패턴 | 구조 | 적합한 경우 |
|------|------|------------|
| **F패턴** | 상단 가로 스캔 → 좌측 세로 스캔. 텍스트 중심 콘텐츠 | 블로그, 뉴스, 문서 — 텍스트가 많은 읽기형 페이지 |
| **Z패턴** | 좌상 → 우상 → 좌하 → 우하 대각선 흐름 | 랜딩 페이지, 마케팅 — 명확한 CTA가 있는 단일 목적 페이지 |
| **카드 그리드** | 동일 크기 카드가 그리드로 배열. 반응형으로 열 수 조절 | 상품 목록, 콘텐츠 피드, 포트폴리오 — 동등한 항목을 탐색 |
| **매거진 레이아웃** | 다양한 크기의 카드/블록이 불규칙하게 배치. 시각적 리듬 | 미디어, 뉴스 — 콘텐츠 중요도에 따라 크기를 차별화 |
| **스플릿 스크린** | 화면을 좌우 또는 상하로 2등분. 각 영역에 대등한 콘텐츠 | 비교, 이중 CTA — 두 가지 선택지를 동등하게 제시 |
| **싱글 컬럼** | 하나의 넓은 열에 콘텐츠를 세로로 배치 | 장문 콘텐츠, 스토리텔링 — 읽기에 집중시키는 페이지 |
| **히어로 + 콘텐츠** | 상단에 대형 비주얼/메시지 + 하단에 세부 콘텐츠 | 랜딩 페이지, 제품 소개 — 첫인상 임팩트 후 상세 정보 |

이 두 레벨은 조합된다. 예를 들어 "마스터-디테일 스크린 패턴 + 좌측 목록은 카드 그리드 레이아웃"처럼.

## 3. 화면 전환/표시 패턴 — 언제 무엇을 쓰나

### 별도 페이지 (Full Page Navigation)
- 작업 맥락이 완전히 전환될 때
- 복잡한 멀티스텝 프로세스 (설정 마법사, 온보딩 플로우)
- 사용자가 이전 화면을 참조할 필요가 없을 때
- 검색 엔진 인덱싱이 필요한 콘텐츠

### 마스터-디테일 (Master-Detail / Split View)
- 목록에서 항목을 선택하면 옆에 상세가 나타나는 패턴
- 여러 항목을 빠르게 훑어봐야 할 때 (이메일, 파일, 이슈 목록)
- 수평 분할: 마스터 목록에 정보가 많을 때 (표 형태)
- 수직 분할: 좌측 목록 + 우측 상세 (메일 클라이언트, Linear, Notion)
- 핵심 장점: 화면 이동 없이 맥락을 유지하며 탐색

### 사이드 패널 / 드로어 (Side Panel / Drawer)
- 모달보다 복잡하지만 별도 페이지까지는 필요 없는 작업
- 메인 콘텐츠를 참조하면서 동시에 부가 작업을 할 때
- 편집, 속성 변경, 코멘트 등 — 맥락을 잃지 않는 것이 중요한 작업
- 예: Figma의 속성 패널, Notion의 페이지 미리보기

### 모달 / 다이얼로그 (Modal / Dialog)
- 사용자를 의도적으로 멈추게 하고 집중시켜야 할 때
- 확인이 필요한 위험한 액션 (삭제, 권한 변경)
- 짧고 독립적인 입력 (이름 변경, 빠른 생성)
- **피해야 할 때**: 에러 메시지, 기능 알림, 온보딩, 반복 작업

### 인라인 확장 (Inline Expansion)
- 테이블 행 확장, 아코디언, 토글 섹션
- 반복적으로 여러 항목의 세부 정보를 확인할 때
- 맥락 전환 비용을 최소화해야 할 때

### 탭 (In-page Tabs)
- 같은 대상에 대한 서로 다른 관점/카테고리를 보여줄 때
- 동시에 볼 필요는 없지만 빠르게 전환해야 할 때
- 탭 수는 5-7개 이하. 그 이상이면 정보구조 재설계 필요

## 4. 제품 유형별 데스크탑 패턴

### 대시보드 / 분석형
데스크탑 대시보드 사용자는 **분석 모드** — 깊이 파고들기, 교차 참조, 패턴 발견이 목적.

- 한 화면에 15-20개 데이터 포인트를 동시 표시 가능
- **위젯 그리드**: 카드 형태의 차트/지표를 그리드로 배치. 사용자가 배치를 커스터마이즈할 수 있으면 좋음
- **드릴다운**: 차트 클릭 → 사이드 패널 또는 인라인 확장으로 상세. 별도 페이지로 이동하면 전체 맥락을 잃음
- **필터**: 상단 또는 좌측 사이드바에 고정. 필터 변경이 전체 대시보드에 즉시 반영
- **비교 뷰**: 나란히 놓기가 가능 — 시간 비교, 세그먼트 비교 등을 split view로

### B2C 소비자 웹
데스크탑에서는 신뢰 구축과 정보 깊이가 모바일보다 중요.

- **랜딩 페이지**: 히어로 → 가치 제안 → 사회적 증거 → CTA. F패턴 또는 Z패턴 읽기 흐름
- **콘텐츠 소비**: 넓은 화면을 활용한 사이드바(목차, 관련 콘텐츠)와 본문 병렬 배치
- **전환 최적화**: 모바일보다 세션이 길고(5-8분) 구매 전환이 높음. 비교, 리뷰, 상세 스펙 등 의사결정 지원 정보를 충분히 제공
- **호버 프리뷰**: 상품/콘텐츠 위에 마우스를 올리면 미리보기 — 클릭 전에 판단할 수 있게

### 생산성 / SaaS 도구
효율과 파워유저 지원이 핵심. Jakob's Law — 사용자는 이미 쓰는 도구와 비슷하게 작동하길 기대.

- **워크스페이스 레이아웃**: 좌측 사이드바(네비게이션) + 중앙(메인 콘텐츠) + 우측 패널(속성/디테일). Notion, Linear, Figma 모두 이 패턴
- **마스터-디테일**: 이슈 목록, 파일 탐색, 메일 등에서 표준
- **커맨드 팔레트**: Cmd+K로 어디서든 빠른 접근. 파워유저의 효율을 극대화
- **키보드 단축키**: 마우스 없이 주요 작업 수행 가능해야 함
- **드래그 앤 드롭**: 정렬, 이동, 구성 변경에 활용
- **멀티 윈도우 / 멀티탭**: 동시에 여러 문서/뷰를 열어놓고 작업

### 콘텐츠 / 미디어형
탐색과 발견, 그리고 깊은 소비의 균형.

- **그리드 레이아웃**: 카드 그리드로 탐색. 화면 폭에 따라 열 수 조절
- **무한 스크롤 vs 페이지네이션**: 탐색형은 무한 스크롤, 목적형은 페이지네이션
- **읽기 뷰**: 본문 폭 제한(60-80자/줄), 양쪽에 여백. 사이드바에 목차나 관련 콘텐츠
- **미디어 뷰어**: 라이트박스/시어터 모드로 콘텐츠에 몰입. ESC로 복귀

### 이커머스
데스크탑은 비교와 확신 부여에 최적화.

- **상품 목록**: 그리드 + 좌측 필터 사이드바. 필터 변경 시 즉시 반영
- **상품 상세**: 좌측 이미지 갤러리 + 우측 정보/구매 영역. 스크롤 시 구매 버튼 sticky
- **비교 기능**: 나란히 놓기가 자연스러움 — 넓은 화면의 최대 장점
- **체크아웃**: 신뢰 요소(보증, 리뷰, 보안 배지) 노출. 모바일보다 폼 필드에 관대하지만, 불필요한 입력은 여전히 제거
- **호버 상태**: Quick View로 목록에서 벗어나지 않고 상품 미리보기

### 관리자 / 백오피스 도구
데이터 밀도가 높고, 효율이 최우선.

- **테이블 중심**: 대량 데이터를 테이블로. 정렬, 필터, 검색, 컬럼 커스터마이즈
- **인라인 편집**: 셀 클릭으로 바로 수정. 매번 상세 페이지로 이동하지 않음
- **벌크 액션**: 멀티셀렉트 + 일괄 작업 (상태 변경, 삭제, 내보내기)
- **CRUD 폼**: 생성/수정은 모달 또는 사이드 패널. 입력이 복잡하면 별도 페이지
- **브레드크럼**: 깊은 계층 구조에서 현재 위치 표시 + 상위로 빠른 이동

## 5. 인터랙션 특성

### 마우스 & 호버
- **호버 상태**: 클릭 가능한 모든 요소에 호버 피드백. 데스크탑의 핵심 어포던스
- **호버 프리뷰**: 링크, 썸네일 위에서 미리보기 — 클릭 비용 없이 판단
- **우클릭 컨텍스트 메뉴**: 파워유저를 위한 빠른 액션 접근
- **드래그 앤 드롭**: 정렬, 파일 업로드, 레이아웃 커스터마이즈

### 키보드
- **단축키**: 주요 액션에 키보드 단축키 제공. 학습 곡선이 있지만 효율 극대화
- **탭 네비게이션**: 폼에서 Tab으로 필드 이동. 논리적 순서로 포커스 이동
- **커맨드 팔레트**: Cmd+K 또는 / 로 빠른 네비게이션/액션. SaaS 표준이 되어감

### 멀티 윈도우
- 사용자가 여러 탭/윈도우를 동시에 열어놓는 것이 일반적
- 각 페이지가 독립적으로 작동해야 함 (상태가 공유되더라도)
- 딥링크 지원: URL이 현재 뷰의 상태를 반영해야 함

## 6. 네비게이션 구조

| 패턴 | 특징 | 적합한 경우 |
|------|------|------------|
| **좌측 사이드바** | 항상 보이는 주 네비게이션. 접을 수 있음 | SaaS, 관리자 도구, 대시보드 |
| **상단 네비게이션** | 수평 메뉴. 섹션이 적을 때 깔끔 | 마케팅 사이트, 콘텐츠 사이트 |
| **상단 + 좌측 조합** | 상단에 제품 전환, 좌측에 섹션 네비게이션 | 복잡한 SaaS (Google Cloud, AWS) |
| **브레드크럼** | 계층 구조에서 현재 위치 표시 | 깊은 카테고리, 파일 시스템 |
| **커맨드 팔레트** | 검색 기반 네비게이션 | 파워유저가 많은 도구형 제품 |

**사이드바 설계 원칙:**
- 폭: 메인 콘텐츠를 압도하지 않으면서 레이블이 잘리지 않는 선
- 텍스트: 14-16px. 그 이하는 가독성 저하
- 구조: 주요 네비게이션은 상단, 설정/계정/로그아웃은 하단에 분리
- 접기: 아이콘만 보이는 축소 모드 지원. 메인 콘텐츠 영역 최대화

---

## 7. 레이아웃 & 그리드

Material Design의 반응형 그리드:
- 데스크탑: 12컬럼
- 태블릿: 8컬럼
- 모바일: 4컬럼

**데스크탑 레이아웃 원칙:**
- 컬럼 폭은 퍼센트 기반 (고정값이 아닌 유동적)
- 최대 콘텐츠 폭 설정 (1200-1440px 일반적) — 초대형 모니터에서 가독성 유지
- 여백(margin)과 거터(gutter)로 영역 간 시각적 분리
- 브레이크포인트에서 컴포넌트가 재배치(reposition), 크기 조정(resize), 또는 교체(swap)될 수 있음
