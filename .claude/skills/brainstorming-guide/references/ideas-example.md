# Ideas Example — 레시피 공유 플랫폼 "쿡로그"

> Feature: 레시피 공유 — 사용자가 자신의 레시피를 등록하고 다른 사람과 공유하는 기능
>
> 아래 아이디어들은 Pain Points와 독립적으로 탐색된 것입니다. Pain Point와의 연결은 이 스킬의 범위 밖에서 수행합니다.

## Idea 1: 음성 + 사진 기반 즉석 기록 — "쿡 메모"

요리 중에 손을 쓰지 않고도 레시피를 기록할 수 있는 모드. 음성 명령으로 사진을 찍고 한마디씩 남기면, 나중에 이것들이 자동으로 레시피 초안으로 조합되는 방식.

- 요리 시작 전에 "쿡 메모 시작"을 누르면 기록 모드 진입. 이후 "사진"이라고 말하면 카메라가 켜지고, 찍은 뒤 "양파 볶다가 갈색 나면 간장 한 바퀴" 같은 음성 메모를 남김
- 요리가 끝나면 시간순으로 사진과 메모가 나열된 "러프 기록"이 만들어짐. 이걸 그대로 올릴 수도 있고, 나중에 편집해서 정돈할 수도 있음
- "대충 남긴 기록"과 "정돈된 레시피" 사이에 중간 단계를 두는 것이 핵심. 완벽하지 않아도 일단 남길 수 있는 허들 낮춤
- 참고: Instagram Stories의 캐주얼한 기록 방식 + 음성 메모 앱의 편의성 조합. Notion AI가 음성 메모를 구조화하는 것과 유사한 접근

## Idea 2: 레시피 버전 관리 — "쿡 히스토리"

같은 요리를 반복하면서 변형한 이력을 자동으로 추적하고, 버전 간 차이를 시각적으로 보여주는 기능.

- 기존 레시피를 기반으로 "다시 만들기"를 누르면, 원본을 복사한 편집 화면이 나옴. 여기서 변경한 부분만 수정하면 새 버전으로 저장됨
- 버전 타임라인이 있어서 "김치찌개 v1 → v2 → v3"처럼 변경 이력이 보임. 각 버전 간의 차이점이 하이라이트됨 (예: v2에서 고춧가루 1큰술 → 0.5큰술로 변경)
- 각 버전에 한줄 메모를 남길 수 있음: "v2: 매운 거 줄이니까 어머니 반응 좋았음", "v3: 참치 넣었더니 국물이 진해짐"
- Git의 커밋 히스토리에서 영감을 받되, 요리하는 사람이 직관적으로 이해할 수 있는 시각적 표현으로 단순화

## Idea 3: 맥락이 살아있는 레시피 카드 — "리얼 레시피"

완성된 레시피 문서가 아니라, 만든 사람의 맥락(브랜드, 도구, 노하우)이 함께 담기는 레시피 포맷.

- 레시피에 사용한 구체적인 제품 정보가 자연스럽게 포함됨. "간장 2큰술" 대신 "샘표 진간장 2큰술 (양조간장 쓰면 좀 더 달아짐)"처럼 대안까지 포함
- 중간 과정의 "이 정도면 됐다" 기준이 사진이나 짧은 영상으로 첨부됨. "반죽이 이 정도 점도가 되면 OK" 같은 시각적 기준점
- 작성자의 코멘트가 단계별로 인라인으로 들어감. 레시피 텍스트와 구분되는 말풍선 형태로 "여기서 센 불로 확 볶는 게 포인트예요" 같은 팁이 표시됨
- 참고: Genius.com에서 가사에 주석을 다는 방식 + 코드 리뷰에서 인라인 코멘트를 다는 방식의 요리 버전

## Idea 4: 나만의 요리 컬렉션 — "쿡북"

기존 카테고리(한식/양식)가 아닌, 사용자가 자유롭게 정의하는 개인 컬렉션 시스템.

- "비 오는 날 메뉴", "15분 완성", "친구 올 때 대접 요리", "다이어트 주간" 같은 사용자 정의 컬렉션을 만들 수 있음
- 하나의 레시피가 여러 컬렉션에 동시에 포함 가능. 태그 시스템이 아니라, Spotify 플레이리스트처럼 의미 단위로 묶는 방식
- 컬렉션을 통째로 공유할 수 있음. "우리 집 일주일 식단"을 컬렉션으로 만들어서 링크 하나로 공유
- 시간이 지나면 AI가 사용 패턴을 분석해서 컬렉션을 자동 제안: "월요일에 자주 만드시는 간단한 요리들, '월요일 간편식'으로 묶어볼까요?"

## Idea 5: 소규모 레시피 서클 — "쿡 그룹"

전체 공개가 아닌, 소수의 사람들과 레시피를 주고받는 프라이빗 그룹.

- 가족, 친구, 동료 등 소규모 그룹을 만들어서 그 안에서만 레시피를 공유. "엄마의 요리" 그룹에 엄마가 레시피를 올리면 가족들만 볼 수 있음
- 그룹 내에서 같은 레시피를 각자 만들어보고 결과를 올리는 "쿡 챌린지" 기능. "이번 주는 다 같이 된장찌개 만들어보기" — 각자의 버전과 결과물 사진이 나란히 보임
- 그룹 레시피북이 자연스럽게 만들어짐. 시간이 지나면 "우리 가족 레시피 50선" 같은 공동 컬렉션이 됨
- 참고: 카카오톡 그룹채팅의 친밀감 + Pinterest 보드의 공동 큐레이션 방식

## Idea 6: 요리 저널 — "쿡 다이어리"

레시피 기록이 아닌, 요리하는 나의 성장 일기. 어떤 요리를 만들었고, 뭘 배웠고, 어떻게 발전했는지를 자동으로 추적.

- 월별 요리 활동 요약: "3월에 12가지 요리를 했고, 처음 시도한 요리 3개, 가장 자주 만든 건 파스타 (4회)"
- 처음 만든 요리와 최근 만든 같은 요리를 나란히 비교: 사진, 사용한 재료, 자기 평가를 시간순으로 보여줌
- "올해 처음 도전한 것들" — 새로운 재료, 새로운 조리법, 새로운 도구를 사용한 기록이 자동으로 모임
- 개인적인 요리 마일스톤: "파스타 10회 달성", "발효 요리 첫 성공", "5일 연속 직접 요리" 같은 작은 성취 알림
- 참고: GitHub 잔디밭(contribution graph)의 요리 버전 + 운동 앱의 성장 기록 방식
