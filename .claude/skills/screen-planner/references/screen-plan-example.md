# Screen Plan — 레시피 공유

- **Flow:** "내가 만든 거 올려야지" — 레시피 공유
- **Platform:** Mobile (iOS/Android)

## Screen Flow

| # | Screen | 한줄 요약 |
|---|--------|----------|
| 1 | 사진 선택 | 요리 사진을 찍거나 갤러리에서 고르는 첫 화면 |
| 2 | 기본 정보 입력 | 요리 이름, 한줄 소개, 자동 카테고리 제안 |
| 3 | 레시피 본문 작성 | 간단 모드(자유 텍스트) 또는 구조 모드(재료+단계) 선택 |
| 4 | 미리보기 & 공개 설정 | 다른 사람에게 보이는 모습 확인, 공개 범위 설정 |
| 5 | 공유 완료 | 완료 확인, 공유 링크, 특정인에게 보내기 |

---

## 공통 요소

이 플로우의 모든 화면에 공통으로 존재하는 요소.

- **하단 탭 바** — 홈, 탐색, + (새 레시피), 내 레시피, 프로필. 현재 플로우에서는 "+" 탭이 활성 상태. 하단 탭은 플로우 진행 중에도 항상 표시되어, 사용자가 작성을 중단하고 다른 섹션으로 이동할 수 있음.
  - 예외: Screen 3(레시피 본문 작성)에서 키보드가 올라오면 하단 탭이 가려짐 — 이 때는 자연스러운 동작이므로 별도 처리 불필요.

- **상단 네비게이션 바** — 좌측에 "X" (닫기) 버튼, 중앙에 단계 표시("1/5", "2/5" 등), 우측에 "임시저장" 텍스트 버튼. 작성 플로우가 모달 성격이므로, 뒤로가기(←)가 아닌 닫기(X)를 사용하여 기존 네비게이션 스택과 분리됨을 표시.
  - "X" 탭 시: 작성 중인 내용이 있으면 "임시저장할까요?" 확인 → 저장 후 이전 화면으로, 또는 삭제 후 이전 화면으로.
  - "임시저장" 탭 시: 현재까지 입력을 저장하고 이전 화면으로 복귀. 내 레시피 탭에서 임시저장 목록으로 접근 가능.

- **진입점** — 홈 FAB(+) 버튼, 내 레시피 탭 → 새 레시피, 또는 다른 사람의 레시피 상세 → "내 버전으로 저장". 진입 경로에 따라 Screen 1의 초기 상태가 달라짐 (사진 프리필 여부).

- **이탈점** — 상단 "X" 닫기, 하단 탭으로 다른 섹션 이동. 어느 화면에서든 이탈 가능하며, 작성 내용이 있으면 임시저장 여부를 확인.

## Screen 1: 사진 선택

**역할:** 레시피 작성의 시작점. "사진부터"라는 진입 방식으로 작성 부담을 낮추는 역할.

**주요 요소:**

- **카메라/갤러리 선택 영역** — 화면 중앙에 크게 배치. 요리 직후의 사용자는 "방금 찍은 사진"이 가장 먼저 하고 싶은 행동이기 때문에, 텍스트 입력이 아닌 사진을 첫 액션으로 제시함. 빈 양식을 마주하는 부담감을 사진 한 장으로 대체하는 것이 핵심.
  - Interaction: 카메라 선택 시 촬영 화면으로 이동, 갤러리 선택 시 사진 선택기 열림
  - Note: 복수 선택 가능 (최대 10장). 첫 장이 대표 이미지가 됨.

- **사진 미리보기 영역** — 선택한 사진들이 가로 스크롤 썸네일로 나타남. 드래그로 순서 변경 가능.
  - Interaction: 개별 사진 탭 시 삭제/크롭 옵션
  - Note: 사진이 0장일 때는 이 영역이 보이지 않고, "사진 없이 레시피만 작성할게요" 텍스트 링크가 하단에 있음. 사진 없이도 진행 가능하지만 권장 경로는 아님.

- **다음 버튼** — 사진이 1장 이상 선택되면 활성화. "벌써 반은 했어요!"라는 느낌을 줄 수 있는 미세한 피드백(프로그레스 표시 등)이 함께 있으면 좋음.
  - Interaction: 탭 시 Screen 2로 이동

**화면 전환:**
- From: 공통 요소의 진입점 참조. 진입 경로에 따라 사진 프리필 여부가 달라짐.
- To: Screen 2 (기본 정보 입력)

**참고:**
- "내 버전으로 저장"으로 진입한 경우, 원본 레시피의 사진이 이미 프리필되어 있음. 사용자는 자기가 직접 찍은 사진으로 교체하거나 그대로 둘 수 있음.
- 이 화면의 핵심 가치는 "작성을 시작하는 허들을 낮추는 것." 첫 화면에서 제목이나 설명을 요구하면 "뭐라고 쓰지..."부터 고민하게 되지만, 사진은 이미 있으니까 그냥 고르기만 하면 됨.

## Screen 2: 기본 정보 입력

**역할:** 레시피의 정체성을 잡아주는 화면. 최소한의 텍스트만 요구하여 흐름을 끊지 않음.

**주요 요소:**

- **선택된 사진 미리보기** — 상단에 대표 사진이 작게 보임. "내가 뭘 올리고 있는지" 맥락을 유지시켜 줌. 작성 중에도 자기가 만든 요리 사진이 보이면 동기 부여가 됨.
  - Interaction: 탭 시 Screen 1로 돌아가서 사진 변경 가능

- **요리 이름 입력 필드** — 필수. 플레이스홀더: "예: 크림 파스타, 된장찌개". 짧고 명확한 이름을 유도.
  - Note: 이름을 입력하면 앱이 자동으로 카테고리를 제안 (아래 요소). 입력 전에는 카테고리 영역이 보이지 않음.

- **자동 카테고리 제안** — 요리 이름 입력 후 나타남. "면 요리"처럼 앱이 추론한 카테고리가 칩(chip) 형태로 표시됨. 사용자는 수락, 변경, 또는 무시할 수 있음.
  - Interaction: 칩 탭 시 카테고리 목록에서 변경 가능, 무시하면 미분류로 저장됨
  - Note: 이 시점에 분류를 강제하지 않는 것이 중요. "일단 올리고 나중에 정리"를 허용해야 요리 직후의 모멘텀을 보존함.

- **한줄 소개 입력 필드** — 선택사항. 플레이스홀더: "예: 15분 만에 뚝딱, 아이들이 좋아하는 맛". 없어도 다음으로 넘어갈 수 있음.

- **다음 버튼** — 요리 이름이 입력되면 활성화.

**화면 전환:**
- From: Screen 1 (사진 선택)
- To: Screen 3 (레시피 본문 작성)

**참고:**
- "내 버전으로 저장"으로 진입한 경우, 요리 이름이 "OO님의 크림 파스타 (내 버전)" 식으로 프리필됨. 사용자가 자유롭게 수정 가능.
- 카테고리 자동 제안이 자연어 처리에 의존하므로, 정확도가 낮을 수 있음. 틀린 제안은 "이 카테고리가 아닌 것 같아요"를 유발할 수 있는데, 무시할 수 있으므로 큰 문제는 아님. 다만 제안이 맞으면 "오, 알아서 해주네" 하는 긍정적 경험이 됨.
