# UX Project Context

## 프로젝트 개요
- 서비스명: Match Table
- 도메인: 맛집/카페 큐레이션 및 예약 (캐치테이블 유사 포지션)
- 플랫폼: 데스크탑/모바일 앱 — 현재 웹 팀 소속
- 핵심 사용자: 맛집·카페에 관심 많은 20–35세
  - 새로운 곳을 발견하고 싶은 사람
  - 데이트·모임·혼밥 등 상황에 맞는 장소를 고르는 사람
  - 방문한 곳을 기록하고 관리하고 싶은 사람
- 현재 단계: 기획 단계 (신규 피처 — AI 취향 분석 + 에이전트)

## 서비스 구조
- 주요 탭: 탐색, 나의 맛집, 매거진, 피드
- 핵심 플로우: 발견 → 큐레이션 → 예약

## 기획 배경
- 경영진이 AI 기능 도입을 강하게 드라이브
- 구체적 방향은 미정인 상태에서 UX팀이 'AI 기능을 어떤 방식으로 넣을지' 기획하는 과제를 받음
- 팀 방향: AI 취향 분석 + 에이전트 기능으로 기존 고객의 예약을 늘리는 방향
- 현재 입맛 프로필과 AI 채팅 피처를 구체화하는 중

## 입맛 프로필 (Taste Profile) — 핵심 신규 피처
- **사내 관심도 높음**: AI를 처음 투입하는 피처라 경영진·조직 전반의 주목을 받고 있음
- **위상**: 메인 GNB에 진입점이 생기는 수준의 중요 피처 — 서비스 전체 내비게이션에 영향
- AI 취향 분석 결과를 사용자에게 시각적으로 보여주고, 이를 기반으로 맞춤 큐레이션·예약을 연결하는 흐름

## 활용 가능한 데이터

### 서비스 내 행동 데이터
- 예약 내역 (카테고리, 가격대, 지역)
- 저장한 맛집 목록
- 작성한 리뷰·별점
- 탐색/검색 기록

### 직접 입력된 데이터
- 기본 프로필에 설정된 선호 카테고리 설정
- 기본 프로필에 설정된 선호 지역 설정

### 간접 추론 데이터
- 매거진·피드에서 저장하거나 오래 본 콘텐츠
