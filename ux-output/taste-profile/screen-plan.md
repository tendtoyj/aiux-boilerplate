# Screen Plan — 입맛 프로필 첫 생성·열람

- **Flow:** "내 입맛이 어떻다고? 한번 볼까" — 첫 프로필 생성·열람
- **Platform:** Desktop Web
- **정보 구조:** 시스템 주도 + 결론 먼저 + 의외성 그루핑

## Screen Flow

| # | Screen | 과업 | 한줄 요약 |
|---|--------|------|----------|
| 1 | 인트로 | "해볼까?" | 기능 소개 + 내 데이터 수치 + 시작 버튼 |
| 2 | 내 입맛 프로필 | "내가 어떤 사람인지 보자" → "어디 가볼까?" | 분석 연출 → 유형 공개 → 인사이트 탐색 (익숙→의외→핵심) → 맞춤 추천 |

### 정보 구조 결정 배경

- **위계 — 결론 먼저:** 유형 타이틀 + AI 비주얼을 먼저 보여줘서 "와" 하는 임팩트 확보. 이후 인사이트는 이 결론의 근거로 탐색.
- **그루핑 — 의외성 기준:** 7개 인사이트를 데이터 유형이 아니라 감정적 역할로 분류. 익숙한 확인("맞아") → 의외한 발견("몰랐는데?") → 나를 관통하는 핵심("결국 이거구나") 순서로 감정 곡선 설계.
- **페이스 — 시스템 주도:** 사용자가 자유 탐색하는 것이 아니라, 시스템이 설계한 순서대로 콘텐츠를 경험. "열어보는 순간의 연출"이라는 플로우 핵심 원칙에 충실.
- **화면 분리 기준 — 과업 전환점:** 정보 그룹이 아니라 사용자의 과업이 바뀌는 지점에서 화면을 나눔. "해볼까?" → "내가 어떤 사람인지 보자 → 어디 가볼까?"
- **추천 배치 — 위아래 분리 + 예고:** 인사이트 탐색의 감정 곡선을 온전히 보존하기 위해 추천은 핵심 인사이트 이후에 배치. 단, 유형 공개 시점에 추천 존재를 한 줄 예고하여 스크롤 동기 확보. 의외성 그루핑의 오름차순 구조(익숙→의외→핵심)가 아래로 갈수록 흥미를 높여 이탈률을 낮추는 역할.

## 공통 요소

모든 Screen에 걸쳐 유지되는 요소. 개별 Screen 정의에서는 반복하지 않음.

- **GNB (Global Navigation Bar)** — 상단에 고정. 기존 탭(탐색, 나의 맛집, 매거진, 피드) + 새로 추가된 입맛 프로필 탭. 이 플로우의 진입점이 GNB의 입맛 프로필 탭이며, 플로우 내 모든 화면에서 GNB가 유지되어 다른 탭으로의 이동이 항상 가능.
  - Note: 분석 연출(로딩) 중에도 GNB를 유지할지, 몰입을 위해 일시적으로 숨길지는 디자인 단계에서 검토. 숨기면 몰입감이 높아지지만, 사용자가 "취소하고 나가고 싶을 때" 탈출구가 없어질 수 있음.

## Screen 1: 인트로

**역할:** GNB의 입맛 프로필 탭을 처음 눌렀을 때 보이는 화면. 분석 결과가 바로 나오지 않고, "이게 뭔데? 해볼까?" 하는 판단을 하는 곳.

**주요 요소:**

- **한줄 프레이밍** — "당신의 예약, 리뷰, 저장 목록이 말해주는 입맛 이야기" 같은 카피. 이 기능이 뭔지 설명하지 않고 한 문장으로 직감하게 만드는 역할. 길게 설명하면 읽지 않음 — "내 데이터로 뭔가 해주는 거구나"만 전달되면 충분.

- **내 데이터 수치** — "예약 47건 · 리뷰 12건 · 저장 83곳" 같은 실제 숫자. 이 수치를 보는 순간 두 가지가 동시에 작동함: "이만큼이나 쌓여 있었네"라는 놀라움 + "이걸로 뭐가 나올까"라는 궁금증. 남의 예시 숫자가 아니라 **내 실제 데이터**여야 효과가 있음.
  - Note: 데이터가 부족한 유저(예약 2건 등)는 이 플로우 대상이 아님. 시나리오가 "데이터가 충분히 쌓인 헤비 유저"를 전제하고 있으므로, 이 화면에서 데이터 부족 케이스는 다루지 않음.

- **시작 버튼** — 사용자가 능동적으로 "시작"을 누르는 행위. 시나리오에서 "자동 시작이 아니라 스스로 열어보겠다는 선택이 기대감을 높임"이라고 명시. 버튼을 누르는 순간이 "선물 뜯기 시작"에 해당.

**화면 전환:**
- From: GNB 입맛 프로필 탭 (첫 진입)
- To: Screen 2 (시작 버튼 클릭 → 분석 연출로 전환)

**참고:**
- 이 화면은 첫 방문에만 보임. 분석이 완료된 이후 재방문 시에는 바로 Screen 2(프로필 결과)로 진입.
- 데스크탑 넓은 화면에서 콘텐츠 양이 적은 편(카피 + 수치 + 버튼). 여백을 의도적으로 활용해서 "시작 전의 고요함"을 만들 수 있는 화면 — 레이아웃은 디자인 단계에서 탐색.

## Screen 2: 내 입맛 프로필

**역할:** 이 플로우의 핵심 화면. 분석 연출로 시작해서, 유형 공개 → 인사이트 탐색 → 맞춤 추천까지 "내가 어떤 사람인지 확인하고, 그에 맞는 곳을 발견하는" 과업 전체를 담당.

### 진입 연출: 분석 중

화면에 진입하면 바로 결과가 나오지 않음. 분석 과정이 보이는 연출이 먼저.

- **데이터 처리 과정 시각화** — "예약 내역 분석 중...", "리뷰 데이터 읽는 중...", "저장 목록 확인 중..." 같이 어떤 데이터를 보고 있는지 사용자가 인지할 수 있어야 함. 시나리오에서 "내 데이터가 진짜 처리되고 있다는 실감"이 없으면 "랜덤 아닌가?" 의심이 든다고 명시.
  - Note: 실제 AI 처리 시간과 무관하게 연출로서의 역할. 너무 짧으면 (1초) "진짜 분석한 거 맞아?" 느낌, 너무 길면 (10초+) 이탈. 적정 체감 시간은 디자인 단계에서 테스트.
- 이 연출이 끝나면 자연스럽게 유형 공개 구간으로 전환됨. 별도 클릭 없이 시스템이 자동 전환.

### 구간 1: 유형 공개 — "나 이런 사람이었어?"

분석 연출이 끝난 직후, 사용자가 처음 마주하는 결과. 이 플로우에서 감정적으로 가장 중요한 지점.

- **유형 타이틀** — "매운맛 모험가" 같이 나를 한 마디로 프레이밍. 이게 있어야 "나 이런 사람이래!" 하는 반응이 가능. 통계 수치가 아니라 서사적 이름이어야 함.

- **AI 생성 비주얼** — 내 데이터 기반으로 만들어진, 사람마다 다른 고유 이미지. UX Values에서 "세상에 나만 가진 것이라는 특별함"과 "공유 욕구 극대화"를 강조. 이 비주얼이 유형 타이틀과 함께 첫 뷰포트에서 작동해야 "와" 반응이 나옴.

- **성격 묘사 + 핵심 수치** — "새로운 맛에 거침없이 도전하지만, 결국 돌아오는 건 익숙한 매운맛"이라는 서사 + "상위 3% 매운맛 선호"라는 팩트. 시나리오에서 이 둘이 어우러져야 임팩트와 납득감이 동시에 생긴다고 명시.

- **추천 예고** — "매운맛 모험가인 당신을 위한 맛집도 준비했어요" 같은 한 줄 카피. 이 시점에서 추천의 존재를 미리 알려두어, 인사이트를 탐색하는 동안 "아래에 추천이 있다"는 기대가 스크롤 동기로 작동.

이 요소들이 **첫 뷰포트 안에서 함께 보여야** 함. 스크롤하기 전에 임팩트가 완성되어야 하고, 여기서 "와" 하는 감정이 이후 인사이트 탐색으로 이어지는 동력이 됨.

### 구간 2: 인사이트 탐색

유형 공개 아래로 스크롤하면 7개 인사이트가 의외성 기준 3그룹으로 전개됨. 그룹 간 감정이 **공감 → 놀라움 → 납득**으로 올라가는 구조 — 아래로 갈수록 더 흥미롭기 때문에 스크롤 동기가 유지됨.

**[익숙한 나] — "맞아, 나 이런 쪽이지"**

사용자가 이미 알고 있거나 예상 가능한 패턴. 결과를 보면서 "아 맞아" 하고 고개 끄덕이는 역할. 첫 인사이트가 너무 의외하면 "이거 틀린 거 아냐?" 의심부터 들 수 있으므로, 익숙한 것으로 먼저 신뢰를 쌓음.

- **카테고리 분포** — 한식, 일식, 카페 등 어떤 카테고리를 얼마나 이용하는지. 대부분의 사용자가 자기 선호 카테고리는 이미 알고 있음. "역시 나는 일식파"라는 확인.
- **가격 레인지** — 평소 예약 가격대 분포. "나 생각보다 가성비파였네" 또는 "확실히 특별한 날에 비싼 데 가는 편이구나" — 본인 감각과 크게 벗어나지 않는 인사이트.
- **자주 가는 동네** — 어느 지역에 집중되어 있는지. 순위형 패턴으로 보여줌 (지도 표현은 시나리오에서 기각).

**[의외의 나] — "어? 이건 몰랐는데"**

본인도 의식하지 못했지만, 데이터를 보면 "생각해보니 그럴 수도 있겠다" 하고 수긍 가능한 패턴. UX Values에서 "틀린 느낌이 아니라 수긍 가능한 의외성"이 핵심이라고 강조.

- **요일·시간대 패턴** — "금요일 저녁은 유독 일식" 같은, 본인도 의식 못 했던 루틴. 시나리오에서 직접 예시로 든 인사이트 — 가장 "오, 진짜?" 반응이 나올 가능성이 높은 유형.
- **숨은 취향 키워드** — 리뷰·저장 데이터에서 뽑아낸, 음식 외에 장소에서 중요하게 여기는 것들. "분위기 좋은", "웨이팅 없는", "조용한" 등. 내가 무의식적으로 반복하는 키워드를 보면 "아, 나 이런 거 중요하게 생각하고 있었구나" 하는 발견.

**[핵심의 나] — "결국 나를 관통하는 건 이거"**

익숙한 것과 의외한 것을 지나서, 내 식사 패턴의 중심축이 되는 인사이트. 유형 타이틀의 근거가 되는 핵심. 이 그룹이 감정적으로 가장 강한 구간이며, 바로 뒤에 이어지는 맞춤 추천으로의 전환 동기를 만듦.

- **맛 성향** — 매운맛, 짠맛, 단맛 등의 밸런스. 내 입맛의 전체 형태를 한눈에 보여주는 인사이트. 시나리오에서 "핵심 인사이트"로 명시.
- **시그니처 메뉴** — 가장 많이 예약한 메뉴 카테고리나 반복 주문 패턴. "파스타만 4번째" 같은 구체적 팩트. 가장 개인적이고 구체적인 데이터.

**인사이트 공통 원칙:**
- 인사이트마다 시각적 형태가 달라야 넘겨보는 리듬이 생김. 전부 같은 차트/카드 형태면 단조로움. 구체적 시각화 형태는 디자인 단계에서 탐색.
- 각 인사이트에 데이터 시각화만이 아니라 짧은 해석 카피가 붙어야 함. "그래서 이게 나한테 어떤 의미지?"가 전달되어야 함.
- 시스템 주도 구조이므로, 사용자가 인사이트를 건너뛰거나 순서를 바꾸지 않음. 위에서 아래로 스크롤하며 설계된 순서대로 경험.

### 구간 3: 맞춤 추천 — "그래서 어디 가볼까?"

핵심 인사이트 직후에 자연스럽게 이어지는 추천 섹션. "나는 이런 사람이구나"를 방금 납득한 상태에서 "그런 당신에게"가 나오므로 맥락 연결이 자연스러움.

- **프로필 기반 맞춤 추천 리스트** — "인기 맛집"이 아니라 내 프로필에서 도출된 맥락별 섹션. "금요일, 불금을 즐기는 당신을 위한 추천 레스토랑", "매운맛 러버 상위 1%, 여기 도전해보는 건 어때요?" 같이 왜 이 식당이 나한테 추천됐는지 맥락이 명시적으로 보여야 함.
  - Note: 추천 소스는 전체 프로필 기반으로 통합. 인사이트별로 추천을 분리하지 않음 (시나리오에서 검토 후 기각).
- 예약 자체는 기존 서비스의 예약 플로우를 그대로 탐. 이 화면의 범위는 추천 리스트를 보여주는 것까지.

### 화면 전환

- From: Screen 1 (시작 버튼 클릭 → 분석 연출로 진입)
- To: 개별 식당 상세 / 기존 예약 플로우 (추천 리스트에서 식당 클릭 시)

### 참고

- 이 화면이 재방문 시의 진입점이 됨 (Screen 1 인트로는 첫 방문만). 재방문 시에는 분석 연출 없이 바로 유형 공개 + 인사이트 + 추천이 보임.
- 데스크탑 넓은 화면 특성상, 유형 영역을 상단에 고정하고 인사이트를 스크롤하는 구성이 가능. 스크롤 중에도 "나는 매운맛 모험가"라는 맥락이 유지되면 인사이트와 유형의 연결감이 강해짐. 구체적 레이아웃은 디자인 단계에서 탐색.
