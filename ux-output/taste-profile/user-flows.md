# User Flows — 입맛 프로필 (Taste Profile)

## Flows

| # | Flow | 한줄 요약 |
|---|------|----------|
| 1 | "내 입맛이 어떻다고? 한번 볼까" — 첫 프로필 생성·열람 | AI 분석 결과를 처음 받아보고, "나 이런 사람이었어?" 하는 발견의 순간 |
| 2 | "이거 친구한테 보여줘야지" — 결과 공유 | 분석 결과 카드를 SNS나 메시지로 공유하고, 받는 사람이 "나도 해볼까" 하게 되는 바이럴 흐름 |
| 3 | "그래서 어디 가면 돼?" — 프로필 기반 추천 탐색 | 분석 결과 위에 얹어진 맞춤 추천을 보고 새로운 맛집을 탐색·예약하는 흐름 |
| 4 | "요즘 나 입맛 바뀐 것 같은데" — 프로필 재방문·변화 확인 | 시간이 지난 후 프로필을 다시 열어보고, 변화된 패턴을 발견하는 흐름 |

## Flow 1. "내 입맛이 어떻다고? 한번 볼까" — 첫 프로필 생성·열람

### 진입 동기

이미 Match Table을 오래 써온 헤비 유저. 예약 내역, 저장 목록, 리뷰, 검색 기록이 충분히 쌓여 있는 사람이다. 어느 날 GNB에 새로운 탭이 생긴 걸 발견하고 "이게 뭐지?" 하면서 눌러본다.

이 플로우의 핵심 감정은 **"나도 몰랐던 나를 발견하는 설렘"**. MBTI 결과를 열어보는 순간, 스포티파이 Wrapped를 재생하는 순간과 같은 기대감. 이 감정을 최대한 살리기 위해 결과를 바로 보여주지 않고, "열어보는 순간"을 연출한다.

### 진입 메뉴 구조

- **GNB 입맛 프로필 탭** — 기존 탭(탐색, 나의 맛집, 매거진, 피드) 사이에 새로 추가된 탭. 기존 유저라면 늘 쓰던 앱 하단에 뭔가 하나가 추가된 걸 자연스럽게 발견하게 됨.

### 시나리오

**1. 발견 — "이게 뭐지?"**

GNB에서 입맛 프로필 탭을 처음 누른 순간. 분석 결과가 바로 나오지 않는다.

이 순간의 역할은 **기대감 형성**. 사용자가 두 가지를 이해하게 만들어야 함:
- "내 데이터를 분석해서 뭔가 보여주는 거구나" — 이 기능이 뭔지 직감적으로 파악
- "내 데이터가 이만큼이나 쌓여 있었네" — 예약 47건, 리뷰 12건, 저장 83곳 같은 내 실제 데이터 수치를 보면서 "이걸로 뭐가 나올까" 하는 궁금증

여기서 사용자가 능동적으로 "시작"을 누르는 행위가 있어야 함. 자동 시작이 아니라, 스스로 열어보겠다는 선택이 기대감을 높임.

**2. 기다림 — "내 데이터를 진짜 보고 있구나"**

시작 후 AI 분석이 진행되는 동안의 경험. 이 구간은 단순한 대기 시간이 아니라 경험의 일부.

별도의 질문이나 입력은 받지 않는다 — 이미 데이터가 충분히 쌓인 유저이므로 추가 정보가 필요 없음.

핵심은 **"내 데이터가 진짜 처리되고 있다"는 실감**. 예약 내역, 리뷰, 저장 목록 등 어떤 데이터가 분석되고 있는지 사용자가 인지할 수 있어야 함. 이 과정이 보여야 결과에 대한 신뢰감("이건 진짜 내 데이터를 본 거다")이 생기고, 그렇지 않으면 "랜덤 아닌가?" 하는 의심이 들 수 있음.

**3. 발견의 순간 — "나 이런 사람이었어?"**

분석이 완료되고 결과를 처음 마주하는 순간. 이 플로우에서 감정적으로 가장 중요한 지점.

이 순간에 전달해야 하는 핵심 요소:
- **나를 정의하는 유형** — "매운맛 모험가" 같이 나를 한 마디로 프레이밍해주는 타이틀. 이게 있어야 "나 이런 사람이래!" 하고 반응할 수 있음.
- **AI가 생성한 나만의 비주얼** — 내 데이터를 기반으로 만들어진, 세상에 나만 가진 이미지. 사람마다 결과물이 전부 다르다는 점이 핵심. 공유 욕구와 직결됨 (Flow 2와 연결).
- **성격 묘사 + 핵심 수치** — "새로운 맛에 거침없이 도전하지만, 결국 돌아오는 건 익숙한 매운맛"이라는 서사와 "상위 3% 매운맛 선호"라는 팩트가 어우러져야 임팩트와 납득감이 동시에 생김.

이 세 요소가 첫 화면에서 함께 작동해야 "와, 되게 신기하다" 반응이 나옴. 배치나 레이아웃은 스크린 플래너에서 탐색.

**4. 탐색 — "오, 이것도 그렇네?"**

첫 인상 이후, 개별 인사이트들을 하나씩 탐색하는 경험. 각 인사이트가 하나의 "오, 진짜?" 발견을 만드는 단위.

다룰 수 있는 인사이트 종류 (7가지):
- **맛 성향** — 매운맛, 짠맛, 단맛 등의 밸런스. 내 입맛의 전체 형태를 한눈에 파악할 수 있는 핵심 인사이트.
- **카테고리 분포** — 한식, 일식, 카페 등 어떤 카테고리를 얼마나 이용하는지.
- **요일·시간대 패턴** — "금요일 저녁은 유독 일식" 같은 본인도 의식 못 했던 루틴.
- **가격 레인지** — 평소 예약하는 가격대의 분포.
- **자주 가는 동네** — 어떤 지역에 집중되어 있는지의 패턴. (지도 표현은 서울 쏠림 시 구 단위/전국 단위 기준이 애매해지므로, 순위형 패턴이 더 적합.)
- **숨은 취향 키워드** — 리뷰·저장 데이터에서 뽑아낸, 음식 외에 장소에서 중요하게 여기는 것들 ("분위기 좋은", "웨이팅 없는", "조용한" 등).
- **시그니처 메뉴** — 가장 많이 예약한 메뉴 카테고리나 반복 주문 패턴. "파스타만 4번째" 같은 구체적 팩트.

경험 원칙:
- 한꺼번에 다 보이는 것이 아니라, **하나씩 발견하는 재미**가 있어야 함.
- 인사이트마다 **시각적 형태가 달라야** 넘겨보는 리듬이 생김. 전부 같은 형태면 단조로움.
- 각 인사이트에 데이터 시각화만 있는 게 아니라 **짧은 해석 카피**가 붙어야 "그래서 이게 나한테 어떤 의미지?"가 전달됨.

구체적인 레이아웃, 시각화 형태, 배치 순서는 스크린 플래너에서 탐색.

**5. 연결 — "그래서 어디 가볼까?"**

인사이트를 다 탐색한 후, 내 프로필 데이터에 근거한 맞춤 추천으로 자연스럽게 이어지는 경험.

추천은 "인기 맛집"이 아니라 내 프로필에서 도출된 맥락별 섹션으로 구성됨:
- "금요일, 불금을 즐기는 당신을 위한 추천 레스토랑"
- "매운맛 러버 상위 1%, 여기 도전해보는 건 어때요?"

**왜 이 식당이 나한테 추천됐는지 맥락이 명시적으로 보이는 것이 핵심.** 프로필 분석 결과 위에 추천이 얹어지기 때문에 "이건 진짜 나한테 맞는 곳이겠다"는 신뢰감이 생기고, 그 신뢰감이 새로운 장소 탐색과 예약으로 이어지는 동기가 됨.

예약 자체는 기존 서비스의 예약 플로우를 그대로 탄다. 이 플로우의 범위는 추천 리스트를 보여주는 것까지.

### 참고

- **"열어보는 순간을 연출"**하는 것이 이 피처의 핵심 설계 원칙. 결과를 바로 보여주면 발견의 재미가 반감됨. 인트로 → 로딩 → 결과의 흐름에서 기대감이 점진적으로 쌓여야 함.
- 로딩 연출에서 데이터 처리 과정을 보여주는 것은 신뢰감 형성에 중요. 실제 처리 시간과 관계없이 연출로서의 역할이 있음.
- AI 생성 이미지는 사람마다 다른 유니크한 결과물이 나와야 공유 욕구가 극대화됨 (Flow 2와 연결).
- 개별 인사이트를 눌렀을 때 인사이트별 맞춤 추천을 보여주는 방식도 검토했으나, 인사이트마다 추천이 달라지면 어색해서 기각. 대신 전체 프로필 기반 추천으로 통합.
- 자주 가는 동네를 지도(히트맵/핀)로 표현하는 방식도 검토했으나, 서울 쏠림 시 구 단위/전국 단위 기준이 애매해지는 문제가 있어 기각.

### 핵심 가치

- **"나 이런 사람이었어?"** — 이 플로우의 전부는 이 한 마디에 담겨 있음. 매일 하던 식사와 예약이 AI 분석을 거쳐 하나의 프로필로 정리되어 눈앞에 보이는 경험. 통계 숫자 나열이 아니라 "당신은 이런 사람이에요"라는 서사가 있어야 함.
- 기대감 → 실감 → 임팩트 → 발견의 재미 → 신뢰감. 이 감정 곡선이 끊기지 않고 자연스럽게 흐르는 것이 이 플로우의 성공 기준.
