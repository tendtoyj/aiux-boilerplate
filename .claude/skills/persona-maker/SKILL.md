# Persona Maker

사용자와 단계별 대화를 통해 목적에 맞는 다수의 UX 페르소나를 도출하고, 최종적으로 `persona.md` 파일을 제공하는 스킬.

## UX Memory Protocol

### 시작 시
1. `ux-memory/project-context.md`가 존재하면 읽어서 프로젝트 맥락을 로드한다.
2. 작업 대상 Feature가 있다면 `ux-output/{feature-name}/`의 기존 산출물(pain-points, ux-values, ideas 등)을 확인하여 페르소나 생성에 참고한다.

### 종료 시
1. 산출물을 `ux-output/{feature-name}/persona.md`에 저장한다.
   - Feature가 지정되지 않은 경우, 사용자에게 어떤 Feature와 연결할지 물어보거나 독립 폴더를 만든다.
2. `ux-memory/activity-log.md`에 실행 기록을 추가한다:
   `| [날짜] | persona-maker | [feature-name] | ux-output/[feature-name]/persona.md | [페르소나 수, 타입] |`
3. `ux-memory/project-context.md`의 "진행 중인 Feature" 테이블을 갱신한다.

---

## General Rules

- 사용자와 대화할 때는 친근하게 "~해요", "~하세요", "~해주세요" 말투를 사용하고 짧게 말한다.
- 모든 대화는 유저가 특별히 다른 언어를 요청하지 않는 한, 한국어로 진행한다.
- 최종결과물을 사용자에게 제시하기 전, 제한사항과 결과물 형식을 모두 지켰는지 확인한 후 제공한다.

## Workflow

### Step 1: 사용자의 목표 정의

목표가 정의되지 않으면 다음 단계로 진행할 수 없다.

**반드시 다음 세 가지 질문으로 시작한다:**

```
1. 이 페르소나로 뭘 하고 싶으세요?
   예시: "피트니스 앱을 만들고 있는데 사용자들이 왜 운동을 포기하는지 알고 싶어서요"
   또는 "온라인 쇼핑몰 상품 페이지를 개선하려고 하는데, 어떤 정보를 먼저 보여줘야 할지 모르겠어요"

2. 타겟 사용자에 대해 이미 알고 있는 정보가 있을까요?
   예시: "20-30대 직장인들이 주 고객인 건 알겠는데, 구체적으론 잘 모르겠어요"
   또는 "고객 설문조사를 했는데 '편리함'을 원한다고 하더라고요. 근데 편리함이 뭔지가..."

3. 가장 중요하게 이해하고 싶은 사용자의 측면은 무엇인가요?
   예시: "사람들이 언제, 왜 우리 앱을 켜는지가 궁금해요"
   또는 "구매 직전에 뭘 망설이는지, 어떤 마음으로 결제 버튼을 누르는지 알고 싶어요"
```

사용자의 대답을 충분히 얻었다면 다음 형식으로 가공하여 제시하고 확인을 받는다:

```
📋 페르소나 제작 계획
목적: [사용자의 목적]
타겟 범위: [타겟 사용자]
핵심 포커스: [핵심 이해 영역]

🎯 만들어볼 페르소나 유형들
- [유형 1]
- [유형 2]
- [유형 3]

📝 각 페르소나에 포함할 내용
- 기본 정보 (나이, 직업, 라이프스타일)
- [목적에 맞는 항목 1]
- [목적에 맞는 항목 2]
- [목적에 맞는 항목 3]
```

확인을 받은 후 "다음을 진행할까요?"로 진행 확인 후 Step 2로 넘어간다.

### Step 2: 페르소나 생성 계획

목표가 설정된 후에만 이 단계로 진입한다.

1. 다음 **페르소나 타입 구분 기준**에 따라 페르소나가 사용자 타입인지 전문가 타입인지 구분한다:

| 타입 | 정의 | 예시 |
|------|------|------|
| **사용자 타입 (User Persona)** | 해당 제품/서비스를 직접 사용하거나 경험하는 사람. 문제를 겪고 있거나 니즈가 있는 당사자 | 피트니스 앱 사용자, 온라인 쇼핑 고객 등 |
| **전문가 타입 (Expert Persona)** | 해당 분야의 전문 지식이나 영향력을 가진 사람. 사용자에게 조언하거나 의사결정에 영향을 주는 역할 | 피트니스 트레이너, 의사, 인플루언서, 컨설턴트 등 |

2. 다음을 포함한 페르소나 생성 계획을 작성한다:
   - 몇 개의 페르소나가 필요할지
   - 각각 어떤 특성으로 구분할지 기준
   - 페르소나별 간단한 프로필을 표로 작성

3. 다음 형식으로 정리하여 사용자에게 제시하고 검토를 받는다:

```
🎯 페르소나 생성 계획

📋 기본 설정
- 목적: [목적]
- 페르소나 개수: N개 (사용자 타입 X개 + 전문가 타입 Y개)

🔍 구분 기준
- 사용자 타입: [기준 A] × [기준 B]
- 전문가 타입: [기준 C] × [기준 D]

📊 페르소나 프로필 (검토용)
| 페르소나 타입 | 이름/나이 | 직업 | 구분 특성 | 주요 관심사 | 핵심 포인트 |
|--------------|----------|------|----------|------------|-------------|
| 사용자 타입 | ... | ... | ... | ... | ... |
| 전문가 타입 | ... | ... | ... | ... | ... |
```

사용자가 최종적으로 계획을 확정해주면 Step 3으로 넘어간다.

### Step 3: 페르소나 생성

이전 단계에서 세운 계획에 맞춰 실제 페르소나를 작성하는 단계.

**1. 생성 옵션 선택**

사용자에게 두 개의 옵션을 설명하고 선택하게 한다:

```
1️⃣ 페르소나를 한 번에 생성
   → 한 번만 확인하면 빠르게 N명의 페르소나를 생성하지만, 풍부함이 부족해요

2️⃣ 한 명씩 확인하며 생성
   → 조금 더 번거롭지만 풍부하고 생생한 페르소나를 원하는 만큼 만들 수 있어요
```

**2. 옵션별 생성 방식**

- **옵션 1 (한 번에 생성)**: 아래 페르소나 생성 원칙에 따라 N개의 페르소나를 한 번에 제공한다. 특히 성별/직업/나이/성향/특성/환경/기술 등에서 다양성이 드러나도록 한다.

- **옵션 2 (한 명씩 생성)**: STP 기법을 통해 타겟팅 우선순위를 고려하여 1개를 먼저 작성한다. 하나의 페르소나가 매우 충분하고 길며 자세한 내용을 포함하도록 한다. 1개 작성 완료 후 "다음 페르소나도 만들어볼까요?"로 확인 후 반복한다.

**3. 페르소나 생성 원칙**

페르소나 타입에 따라 포함할 섹션이 다르다:

#### 사용자 타입 페르소나 구성

참고 자료: `references/user-persona-examples.md`를 참고하여 비슷한 톤으로 구성한다.

포함 섹션: **프로필, 특성, 환경, 기술 관계성, User Journey Map**
(전문성 섹션은 포함하지 않는다)

- Pain Point를 매우 상세하게 작성한다. 구체적이고 현실적인 3가지 주요 불편사항을 예시를 포함해 매우 상세하게 기술한다.
- Pain Point는 하나당 3줄 이상, 구체적이고 개연성 있게 예시를 들어 설명한다.

#### 전문가 타입 페르소나 구성

참고 자료: `references/expert-persona-examples.md`를 참고하여 비슷한 톤으로 구성한다.

포함 섹션: **프로필, 전문성, 특성(Pain Point 제외), 환경, 기술 관계성**
(User Journey Map은 포함하지 않는다, 특성에서 Pain Point는 포함하지 않는다)

- 전문성을 충실한 수준으로 강조한다. 구체적이고 현실적인 분야, 강점을 포함해 매우 상세하게 기술한다.

#### 공통 페르소나 항목 상세

**1. 프로필**
- 이름: 자연스럽고 현실적인 한국 이름 사용
- 나이/성별: 다양한 연령대와 성별 분포 고려
- 직업: 목표 맥락과 연관된 구체적 직업 (일반적 직업명 대신 구체적 역할 포함)
- 성격/성향/취향: 다면적 성격 특성과 취향으로 페르소나에 깊이 부여

**2. 전문성** (전문가 타입만 해당)
- 전문분야: 구체적인 전문 영역과 강점 분야 기술
- 경력: 연차 및 주요 경력 경로와 성과 포함 (실제 존재하는 대학과 회사)
- 학력/이력: 교육 배경과 주요 이력 사항 (국내 SKY 대학 / 네카라쿠배당토, 해외 Ivy league, FAANG, 주요 컨설팅 회사, 글로벌 Top 50 기업 등)
- 기술적 강점: 특정 도구, 언어, 방법론에 대한 전문성
- 성격적 강점: 창의성, 분석력, 공감능력, 의사소통 능력 등
- 경험적 강점: 특정 문제 해결 경험, 산업 지식, 고유한 통찰력
- 지식 강점: 특화된 분야 지식, 학제간 지식의 융합 능력

**3. 특성**
- 한마디: 페르소나의 핵심 가치관이나 성향을 압축한 문장
- 목표: 서비스/제품 사용의 명확한 단기/장기 목표
- 서비스 이용: 사용 빈도, 선호 기능, 사용 패턴 등 구체적 묘사
- 행동패턴: 일상에서의 의사결정 패턴과 행동 특성
- Pain Point (사용자 타입만 해당)

**4. 환경 (맥락적 배경)**
- 거주환경: 주거 형태, 가족 구성, 라이프스타일 등
- 사회환경: 사회적 관계, 커뮤니티 참여, 직장 환경 등

**5. 기술 관계성**
- 친숙도: 기술 활용 수준과 디지털 리터러시 정도
- 수용도: 새로운 기술이나 서비스에 대한 태도와 적응 속도

**6. User Journey Map** (사용자 타입만 해당, 표로 제시)
- [Awareness, Consideration, Purchase, Retention] x [Touch Point / Pain Point / Solution]

#### 공통 품질 기준

- 충분히 디테일한 정보들로 개연성 있게 구성한다.
- 가짜같지 않도록, 너무 완벽한 페르소나가 아니라 현실적으로 구성한다.
- 페르소나를 여러 개 만들 때, 성별/직업/나이/성향/특성/환경/기술 등에서 다양성이 드러나도록 한다.
- 기본적으로 3개의 페르소나를 만들며, 사용자가 원하면 더 만들거나 단계를 건너뛸 수 있다.

### Step 4: 최종 파일 제공

- 모든 페르소나를 합쳐 `persona.md` 파일로 작성하여 제공한다.
