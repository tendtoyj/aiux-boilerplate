# Screen Plan — 첫 프로필 생성·열람

- **Flow:** "내 입맛이 어떻다고? 한번 볼까" — 첫 프로필 생성·열람
- **Platform:** 웹 (Desktop)
- **Date:** 2026-04-08

## Input 분석 요약

### 결정된 것
- 큰 흐름: 발견(인트로) → 기다림(로딩 연출) → 발견의 순간(결과) → 탐색(인사이트) → 연결(추천)
- 진입점: GNB에 새로 추가된 입맛 프로필 탭
- 결과의 핵심 3요소: 유형 타이틀 + AI 생성 비주얼 + 성격 묘사·핵심 수치 — 함께 작동해야 "와" 반응
- 추가 입력 없음 (이미 데이터가 충분한 헤비 유저 대상)
- 인사이트 7가지: 맛 성향, 카테고리 분포, 요일·시간대 패턴, 가격 레인지, 자주 가는 동네, 숨은 취향 키워드, 시그니처 메뉴
- 통합 추천 (인사이트별 추천은 검토 후 기각)
- 지도 표현 기각 → 순위형 패턴
- 플로우 범위: 추천 리스트까지. 예약은 기존 플로우 활용

### 긴장 관계 & 선택한 방향
- **긴장**: "결론의 임팩트" vs "발견의 재미" — 결과를 먼저 보여주면 인사이트가 부수적 경험이 되고, 인사이트를 먼저 쌓으면 결론까지 인내가 필요
- **선택한 방향**: 임팩트 우선 + 자유 탐색 (안 A)
  - 결론(유형 결과)을 먼저 터뜨려서 "와" 반응을 확보하고, 인사이트는 데스크탑의 넓은 화면을 활용해 대시보드 형태로 자유롭게 탐색
  - 이유: 데스크탑에서 순차 공개보다 한눈에 펼쳐놓고 골라보는 것이 플랫폼 특성에 맞고, 임팩트 속도와 이탈 위험 측면에서 유리

## Screen Flow

| # | Screen | 한줄 요약 |
|---|--------|----------|
| 1 | 첫 만남 — "이게 뭐지?" | 입맛 프로필 탭 첫 진입, 내 데이터 현황 확인 + 분석 시작 |
| 2 | 분석 중 — "내 데이터를 보고 있구나" | AI가 내 데이터를 처리하는 과정을 보여주는 로딩 연출 |
| 3 | 내 입맛 프로필 — "나 이런 사람이었어?" | 유형 결과(히어로) + 인사이트 대시보드 + 맞춤 추천이 한 페이지에 |

- **Screen 1, 2**는 프로필이 아직 생성되지 않았을 때만 보이는 일회성 화면. 한번 분석이 완료되면 이후에는 항상 Screen 3으로 바로 진입.
- **Screen 3**은 이 플로우의 메인 화면. 하나의 긴 페이지 안에 세 개 섹션:
  - 히어로 섹션: 유형 타이틀 + AI 비주얼 + 성격 묘사 (뷰포트 가득)
  - 인사이트 섹션: 7가지 인사이트 카드 그리드 (스크롤 다운)
  - 추천 섹션: 프로필 기반 맞춤 추천 (페이지 하단)

---

## 공통 요소

이 플로우의 모든 화면에 공통으로 존재하는 요소.

- **상단 GNB** — Match Table의 주요 탭(탐색, 나의 맛집, 입맛 프로필, 매거진, 피드) + 로고, 검색, 알림, 프로필/계정. 이 플로우에서는 "입맛 프로필" 탭이 활성 상태. 사용자가 입맛 프로필이 Match Table 서비스의 일부라는 맥락을 유지하면서, 어느 화면에서든 다른 섹션으로 이동할 수 있음.

- **진입점** — GNB 입맛 프로필 탭. 프로필 미생성 상태면 Screen 1로, 생성 완료 상태면 Screen 3으로 진입.

- **이탈점** — GNB 다른 탭 클릭으로 어느 화면에서든 이탈 가능. Screen 2(분석 중)에서 이탈 후 복귀 시, 분석 완료면 Screen 3으로, 진행 중이면 Screen 2로 복귀. Screen 3 추천 섹션에서 레스토랑 클릭 시 기존 예약 플로우로 이동(이 플로우 범위 밖).

- **로그인 상태 전제** — 이미 로그인한 헤비 유저 대상. 상단 바에 프로필 아바타/닉네임 표시.

## Screen 1: 첫 만남 — "이게 뭐지?"

**역할:** GNB에서 입맛 프로필 탭을 처음 눌렀을 때 보이는 화면. "내 데이터를 분석해주는 거구나"를 직감적으로 파악하게 하고, "시작" 버튼을 누르고 싶은 기대감을 만드는 것이 이 화면의 전부.

**주요 요소:**

- **헤드라인 + 서브카피** — 이 기능이 뭔지를 한눈에 알려주는 짧은 메시지. 1-2문장 이내로 "내 데이터를 분석해서 나만의 입맛 프로필을 만들어준다"는 것을 직감적으로 전달.
  - Why: 사용자가 "이게 뭐하는 건지" 3초 안에 파악해야 다음 행동(시작 누르기)이 나옴. 이해하지 못하면 바로 이탈.

- **내 데이터 수치 하이라이트** — "예약 52건 · 리뷰 23건 · 저장 91곳" 같이 내 실제 데이터 개수를 보여주는 영역. 숫자가 크게, 눈에 띄게 표시됨.
  - Why: "내 데이터가 이만큼이나 있었네" 하는 실감이 "이걸로 뭐가 나올까?" 하는 궁금증으로 전환됨. 범용적인 기능 소개가 아니라 **나의** 데이터라는 개인화된 맥락이 기대감의 핵심.
  - Interaction: 별도 인터랙션 없음. 시각적 임팩트 역할.

- **분석 시작 버튼** — "내 입맛 알아보기" 같은 CTA. 화면에서 가장 눈에 띄는 액션.
  - Why: 자동 시작이 아니라 사용자가 스스로 시작을 선택하는 행위가 기대감을 높임. MBTI 테스트에서 "시작하기"를 누르는 순간과 같은 역할.
  - Interaction: 클릭 시 Screen 2(분석 중)로 전환.

- **배경 비주얼/분위기** — 음식·맛집과 관련된 분위기 있는 배경이나 일러스트가 화면의 톤을 잡아줌. 단, 결과 화면(Screen 3)의 AI 비주얼보다 시각적으로 절제되어야 함 — 여기가 클라이맥스가 아니므로.
  - Why: 기대감 형성에는 감성적 분위기도 기여. 데이터 수치만 덜렁 있으면 "분석 도구" 느낌이 나지만, 분위기가 있으면 "재미있는 경험이 될 것 같다"는 인상을 줌.

**화면 전환:**
- From: GNB 입맛 프로필 탭 (프로필 미생성 상태에서만)
- To: Screen 2 (분석 시작 버튼 클릭 시)

**참고:**
- 이 화면은 프로필이 한 번 생성되면 다시 보이지 않는 일회성 경험.
- 데스크탑 화면 중앙에 적절한 폭으로 콘텐츠를 모아서, 넓은 화면에서 시선이 분산되지 않게 함. 히어로 + 콘텐츠 레이아웃 패턴.
- 데이터가 부족한 유저가 이 화면에 올 경우의 엣지 케이스는 시나리오상 범위 밖 — 이 플로우는 데이터가 충분한 헤비 유저 전제.

## Screen 2: 분석 중 — "내 데이터를 보고 있구나"

**역할:** 시작 버튼을 누른 후, AI 분석이 진행되는 동안의 경험. 단순한 대기 화면이 아니라 "내 데이터가 진짜 처리되고 있다"는 실감을 통해 결과에 대한 신뢰감을 형성하는 화면.

**주요 요소:**

- **데이터 처리 과정 시각화** — "예약 내역 분석 중...", "리뷰 키워드 추출 중...", "저장 목록 패턴 분석 중..." 처럼 어떤 데이터가 분석되고 있는지 단계별로 보여줌. 각 단계가 순차적으로 나타나면서 진행감을 줌.
  - Why: 이 과정이 보여야 결과에 대한 신뢰감("이건 진짜 내 데이터를 본 거다")이 생김. 그렇지 않으면 "그냥 랜덤 아닌가?" 하는 의심이 들 수 있음.
  - Interaction: 별도 인터랙션 없음. 사용자는 지켜보는 경험.
  - Note: 실제 처리 시간과 관계없이 연출로서의 역할이 있음. 너무 빨리 끝나면 "진짜 분석한 거 맞아?" 느낌이 들 수 있으므로, 최소 연출 시간이 필요.

- **프로그레스 표시** — 전체 진행률을 보여주는 요소(퍼센트, 프로그레스 바, 단계 카운터 등). "끝이 있다"는 것을 알려줘서 이탈을 방지.
  - Why: 진행 상황이 보이지 않으면 "얼마나 더 기다려야 하지?" 하는 불안이 생기고 이탈할 수 있음. 끝이 보이면 기다릴 수 있음.

- **내 데이터 스니펫** — 분석 중 실제 내 데이터의 조각이 잠깐씩 보임. 예: "최근 예약: 스시 오마카세 (2주 전)", "리뷰에서 자주 쓴 단어: 분위기, 조용한". 데이터가 흘러가듯 나타났다 사라지는 연출.
  - Why: "아, 내 그 리뷰도 보고 있구나" 하는 실감. 추상적인 "분석 중..."보다 내 구체적 데이터가 보이면 몰입도가 올라감.
  - Note: 민감한 데이터가 아닌, 사용자가 이미 직접 입력한 공개 데이터만 노출.

**화면 전환:**
- From: Screen 1 (분석 시작 버튼 클릭)
- To: Screen 3 (분석 완료 시 자동 전환)

**참고:**
- 이 화면도 일회성. 프로필 생성 후에는 다시 보이지 않음.
- 분석 중 이탈 후 복귀 시 — 분석이 완료되었으면 Screen 3으로, 아직 진행 중이면 이 화면의 현재 진행 상태로 복귀.
- 데스크탑에서도 화면 중앙에 콘텐츠를 집중 배치. 이 순간은 "기다리면서 지켜보는" 경험이므로 시선이 분산되지 않아야 함.

## Screen 3: 내 입맛 프로필 — "나 이런 사람이었어?"

**역할:** 이 플로우의 메인 화면. 분석 완료 후 처음 도달하며, 이후 입맛 프로필 탭을 누를 때마다 항상 이 화면으로 진입. 하나의 긴 페이지 안에 세 섹션으로 구성.

### 섹션 1: 히어로 — 유형 결과

첫 뷰포트를 가득 채우는 영역. 이 플로우에서 감정적으로 가장 중요한 지점 — "나 이런 사람이었어?" 반응이 터지는 순간.

**주요 요소:**

- **유형 타이틀** — "매운맛 모험가" 같이 나를 한 마디로 프레이밍해주는 이름. 이 화면에서 가장 먼저 눈에 들어와야 하는 요소.
  - Why: "나 이런 사람이래!" 하고 반응할 수 있는 핵심 장치. MBTI에서 "ENFP"를 보는 순간과 같은 역할. 타이틀이 없으면 결과가 데이터 리포트가 되어버림.

- **AI 생성 비주얼** — 내 데이터를 기반으로 만들어진, 세상에 나만 가진 이미지. 사람마다 결과물이 전부 다름.
  - Why: 텍스트로 "매운맛 러버입니다"라고 보여주는 것과, 나만의 독특한 비주얼이 생성되는 것은 감정적 임팩트가 완전히 다름. 유니크한 비주얼이 공유 욕구를 극대화 (Flow 2와 연결).
  - Note: 히어로 영역에서 비주얼이 충분한 공간을 차지해야 임팩트가 살아남. 작은 아이콘 수준이면 효과 반감.

- **성격 묘사** — "새로운 맛에 거침없이 도전하지만, 결국 돌아오는 건 익숙한 매운맛" 같은 서사적 텍스트. 나를 읽어주는 듯한 느낌.
  - Why: 타이틀이 "라벨"이라면, 성격 묘사는 "해석". 이 텍스트를 읽으면서 "맞아, 나 확실히 이런 쪽이지" 하는 공감이 생기거나, "어? 나 이랬어?" 하는 의외성을 느끼게 됨. 둘 다 사용자를 결과에 몰입하게 만드는 힘이 있음.

- **핵심 수치** — "상위 3% 매운맛 선호", "올해 예약 52건" 같은 팩트 기반 수치. 성격 묘사와 어우러져 임팩트와 납득감이 동시에 생김.
  - Why: 서사만 있으면 "바넘 효과 아닌가?" 의심이 들 수 있고, 수치만 있으면 데이터 리포트가 됨. 서사 + 팩트가 함께 있어야 "와, 되게 신기하다" 반응이 나옴.

- **스크롤 유도 요소** — 히어로 아래에 더 많은 콘텐츠가 있다는 것을 알려주는 시각적 힌트. 화살표, 텍스트("내 인사이트 더 보기"), 또는 아래 섹션의 상단이 살짝 보이는 형태.
  - Why: 히어로에서 만족하고 이탈하는 것을 방지. "아래에 더 재밌는 게 있다"는 것을 알려줘서 인사이트 섹션으로 자연스럽게 유도.

---

### 섹션 2: 인사이트 대시보드 — "오, 이것도 그렇네?"

히어로 아래로 스크롤하면 나타나는 영역. 7가지 인사이트가 카드 형태로 펼쳐져 있어서, 사용자가 관심 가는 것부터 자유롭게 탐색.

**주요 요소:**

- **섹션 헤드라인** — 히어로의 유형 결과가 어떤 근거에서 나왔는지 탐색하는 영역이라는 맥락 전환을 알려주는 짧은 도입 문구.
  - Why: 히어로에서 바로 카드 그리드로 넘어가면 "이건 뭐지?" 하는 단절이 생길 수 있음. 한 줄이 "여기서부터는 상세 탐색이에요"라는 전환 역할.

- **인사이트 카드 그리드** — 7가지 인사이트가 각각 하나의 카드로 표현됨. 데스크탑 화면 폭을 활용해 한 줄에 3-4개씩 배치. 각 카드는 접힌 상태(요약)로 먼저 보이고, 클릭하면 상세가 열림.
  - Why: 대시보드 형태로 한눈에 전체를 보여주는 것이 이 방향(안 A)의 핵심. "내 데이터가 이렇게 다양한 걸 말해주는구나" 하는 풍성함을 한 화면에서 느낄 수 있음.
  - Interaction: 카드 클릭 시 상세 내용 확장.
  - Note: 카드마다 시각적 형태가 달라야 넘겨보는 리듬이 생김. 각 인사이트의 데이터 성격에 맞는 시각화.

- **카드 접힌 상태 (요약)** — 각 카드가 그리드에서 보여주는 기본 정보. 인사이트 이름 + 미니 시각화 + 짧은 해석 카피 한 줄.
  - Why: 접힌 상태만으로도 "오, 이것도 봐야겠다" 하는 흥미를 유발해야 클릭이 일어남. 타이틀만 있으면 동기가 약함.

- **카드 펼친 상태 (상세)** — 카드를 클릭하면 열리는 상세 뷰. 큰 시각화 + 데이터 해석 카피 + 구체적 수치/패턴. 사이드 패널이나 인라인 확장으로 열되, 나머지 카드 맥락을 완전히 잃지 않는 방식.
  - Why: 인사이트 하나하나가 "오, 진짜?" 하는 발견을 만드는 단위. 시각화만 있으면 "그래서?" 이고, 해석 카피가 붙어야 "이게 나한테 어떤 의미인지"가 전달됨.
  - Interaction: 닫기 또는 다른 카드 클릭으로 전환. 한 번에 하나의 카드만 펼쳐져서 포커스가 분산되지 않도록.

**7가지 인사이트 카드 구성:**

| # | 인사이트 | 핵심 전달 | 시각화 힌트 |
|---|---------|----------|------------|
| 1 | 맛 성향 | 매운맛·짠맛·단맛 등의 밸런스. 내 입맛의 전체 형태 | 레이더/방사형 차트 |
| 2 | 카테고리 분포 | 한식·일식·카페 등 어떤 카테고리를 얼마나 | 바 차트 또는 도넛 차트 |
| 3 | 요일·시간대 패턴 | "금요일 저녁은 유독 일식" 같은 무의식적 루틴 | 히트맵 또는 타임라인 |
| 4 | 가격 레인지 | 평소 예약하는 가격대의 분포 | 분포 그래프 |
| 5 | 자주 가는 동네 | 어떤 지역에 집중되어 있는지 (순위형) | 순위 리스트 + 비율 바 |
| 6 | 숨은 취향 키워드 | 리뷰·저장에서 뽑아낸 "분위기 좋은", "조용한" 등 | 워드 클라우드 또는 태그 |
| 7 | 시그니처 메뉴 | 가장 많이 예약한 메뉴·반복 주문 패턴 | 아이콘 + 횟수 강조 |

### 섹션 3: 맞춤 추천 — "그래서 어디 가볼까?"

인사이트 대시보드 아래로 스크롤하면 나타나는 영역. 내 프로필 데이터에 근거한 맞춤 추천으로 자연스럽게 이어지는 경험.

**주요 요소:**

- **섹션 전환 메시지** — 인사이트 탐색(분석 경험)에서 추천 탐색(액션 경험)으로 모드가 전환됨을 알려주는 짧은 도입 문구.
  - Why: 인사이트 카드 그리드 바로 아래에 레스토랑 목록이 나오면 "갑자기 광고?" 같은 느낌이 들 수 있음. 전환 메시지가 "이건 분석 결과를 기반으로 한 추천이에요"라는 맥락을 연결해줌.

- **맥락별 추천 섹션** — "인기 맛집" 같은 범용 추천이 아니라, 내 프로필에서 도출된 맥락이 섹션 제목이 됨. 예: "금요일 불금을 즐기는 당신을 위한 추천 레스토랑", "매운맛 러버 상위 1%, 여기 도전해보는 건 어때요?"
  - Why: 왜 이 식당이 나한테 추천됐는지 맥락이 명시적으로 보이는 것이 핵심. 프로필 분석 위에 추천이 얹어지기 때문에 "이건 진짜 나한테 맞는 곳이겠다"는 신뢰감이 생기고, 그 신뢰감이 탐색과 예약으로 이어지는 동기가 됨.
  - Note: 섹션 수는 2-4개 정도. 너무 많으면 "추천 피로"가 생기고, 너무 적으면 프로필 분석이 풍부하게 활용되지 않는 느낌.

- **레스토랑 카드** — 각 추천 섹션 안에 레스토랑이 가로 스크롤 또는 카드 그리드로 배치. 대표 이미지, 이름, 카테고리, 가격대, 평점 등 기존 서비스에서 사용하는 레스토랑 카드 포맷을 그대로 활용.
  - Why: 사용자가 이미 탐색 탭에서 익숙하게 보던 형태 그대로여야 "여기 눌러서 예약하면 되겠다"는 인지가 바로 됨.
  - Interaction: 카드 클릭 시 기존 예약 플로우로 이동. 호버 시 Quick View로 미리보기 가능.

- **추천 이유 태그** — 각 레스토랑 카드에 "내 프로필 기반" 추천 이유가 짧은 태그로 붙음. 예: "당신의 매운맛 취향", "금요일 저녁 패턴 기반".
  - Why: 추천 섹션 제목만으로는 개별 레스토랑과 내 프로필의 연결이 느슨할 수 있음. 카드 레벨에서도 "이게 왜 나한테 추천됐는지"가 보이면 신뢰감이 한 층 더 올라감.

**화면 전환:**
- From: 인사이트 대시보드 섹션에서 스크롤 다운
- To: 레스토랑 카드 클릭 시 기존 예약 플로우로 이동 (이 플로우의 범위 밖)

**참고:**
- 추천 섹션은 이 플로우가 비즈니스 성과(예약 전환율)와 직결되는 지점. 입맛 프로필의 재미로 유저를 끌어들이고, 자연스럽게 "그래서 여기 가보세요"가 얹어지는 흐름.
- 예약 자체는 기존 서비스의 예약 플로우를 그대로 탐. 이 플로우의 범위는 추천 리스트를 보여주는 것까지.
- 추천 데이터가 아직 없거나 부족한 경우의 엣지 케이스 — 이 플로우는 데이터 충분한 헤비 유저 전제이므로, 추천 결과도 충분할 것으로 가정.
